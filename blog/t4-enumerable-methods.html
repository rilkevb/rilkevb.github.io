Appreciating Enumerable#group_by

10/30/14

The Ruby Enumerable mixin includes many extremely useful methods. Any Ruby class which includes the each method gains access to the Enumerable methods.

I intend to slowly write about many of these methods but the first one I want to focus on is Enumerable#group_by. The general structure of group_by is:

collection.group_by { |obj| block }

When it is called, the group_by method groups the collection according to the result of the block. The method then returns a hash where the keys are the result of the logic block and the values for each key are those members of the collection which returned that key.

For example, imagine you had an array of 9 integers and you wanted to know what the result of performing modulo 5 on each one was. You could use group_by like so:

num = [1,2,3,4,5,6,7,8,9]
(num).group_by{|i| i%5 }

RESULT:

{1=>[1, 6], 2=>[2, 7], 3=>[3, 8], 4=>[4, 9], 0=>[5]}

From the returned hash we can see that 1%5 and 6%5 equal 1, 2%5 and 7%5 equal 2, etc..

We can use group_by to group the results of all sorts of mathematical manipulations. We can also use it return other types of statements. Imagine we want to know which words of a particular quote contain a particular substring and then group them according to this criteria. All we need to do is:

quote = ["to","make","the","world","work","for","100%","of","humanity"]
(quote).group_by{|i| i.include?('akâ€™)}

RETURNS:

{false=>["to", "the", "world", "work", "for", "100%", "of", "humanity"], true=>["make"]}

And now we know that in the given quote, "make" is the only word which contains the substring "ak".

These examples are contrived but it is easy to imagine querying large data sets where group_by would be very useful.
