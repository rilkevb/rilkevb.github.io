Arrays and Hashes in Ruby

10/25/14

In Ruby an array is an ordered list of items. Every slot in the list points to a particular object or value and you can change what the slot points to. (1) Like in other computer languages Ruby arrays are zero indexed. However, unlike in older languages, Ruby arrays are default dynamically resizable. Also, should the programmer index outside an array in Ruby, the language returns 'nil' rather than crashing and/or doing something worse. Arrays are a useful data structure when the sequence of the stored values is important or when it is crucial to have the stored values integer-indexed. (2)

If the programmer doesn't need or want integer indexing then Ruby hashes might be a more appropriate data structure to use. In Ruby, a hash is "a dictionary-like collection of unique keys and their values" (3) Instead of using integers for indexing, a hash can use any object type:

favorite_foods = {'Al'=>'pie','Ed'=>'tofu'}

In this example, when the key 'Al' is referenced, the value 'pie' will be returned. While this is the standard structure and behavior of a 'dictionary' data structure, Ruby calls it a 'hash' because the underlying sorting and retrieval is being run through a hash function. (4)
