Ruby Classes and the Collatz Conjecture

11/7/14

Ruby is an object oriented programming(OOP) language and because of this it is dominated by classes. Classes are extensible templates used to create other code objects. A class can store basic information (attributes) and ways of doing things (methods). A general class is used to create specific objects (also called "instances of the class"). A class is like an apple pie recipe. It contains lists of ingredients (attributes) and instructions for what to do (methods). If you take this recipe and make a particular apple pie then that specific apple pie is like an object or an "instance of the class".

Classes are a natural choice for organizing programs(1) when the programmer wants to group specific data and functions into one unit. If you are modelling a physical object it makes sense to put all its attributes and the actions it can perform into one class. Or, if you are building a collection of tools to analyze or present an abstract problem it can make sense to group all of them into one class.

An example of using classes in this second manner is this class which I wrote: (Please pardon the formatting; I'll make it easier to read later.)

class Collatz_Conjecture

attr_accessor :initial_number

def initialize(number)
@initial_number = number
end


def explanation
puts "The Collatz_Conjecture is named after the mathematician Lothar Collatz who proposed it in 1937."
puts "It is a mathematical conjecture which states:"
puts ''
puts "Take any natural number X. If X is even, divide it by 2. If X is odd, multiply it by 3 and add 1."
puts "Repeat this process indefinitely. The conjecture is that no matter what number you start with, "
puts "you will always eventually reach 1. (wikipedia)"
puts " "
puts "The sequence of numbers produced by executing the above algorithm is often called a 'hailstone sequence'"
puts "because the numbers go up and down erratically like hail in a thundercloud before it reaches the ground."
puts ""
end

def print_numeric_sequence
num = @initial_number
puts "The Numeric Sequence for the Given Value: "
output = [num]
until num == 1
if num%2 == 0
num = num/2
else
num = ((num * 3) + 1)
end
output << num
end
output
end

def print_graph_approximation
num = @initial_number
puts "An Approximation of the Hailstone Sequence for the Given Value:"
print ('-' * num)
puts " #{num}"
until num == 1
if num%2 == 0
num = num/2
else
num = ((num * 3) + 1)
end
print ('-' * num)
puts " #{num}"
end
end

def proof
puts "'Mathematics is not yet ready for such problems.' -Paul Erdos"
end
end

The code is a little long but basically it allows a person to see and play around with the Collatz Conjecture and resulting Hailstone Sequences.

In case you are wondering what the Collatz Conjecture is, all you need to do is create a new object (pass it any positive integer you want) and call the explanation method.

attempt1 = Collatz_Conjecture.new(17)
p attempt1.explanation

This returns:

The Collatz_Conjecture is named after the mathematician Lothar Collatz who proposed it in 1937. It is a mathematical conjecture which states:

Take any natural number X. If X is even, divide it by 2. If X is odd, multiply it by 3 and add 1.
Repeat this process indefinitely. The conjecture is that no matter what number you start with,
you will always eventually reach 1. (wikipedia)

The sequence of numbers produced by executing the above algorithm is often called a 'hailstone sequence'
because the numbers go up and down erratically like hail in a thundercloud.

If you want to see the numeric sequence resulting from the initialization value you passed, just type:

p attempt1.print_numeric_sequence

For this example, using 17 as the initial value, it returns:

[17, 52, 26, 13, 40, 20, 10, 5, 16, 8, 4, 2, 1]

If you want a more graphical representation of the sequence you can type:

attempt1.print_graph_approximation

And it will print a series of labelled bars to the console to help better visualize the sequence:

----------------- 17
---------------------------------------------------- 52
-------------------------- 26
------------- 13
---------------------------------------- 40
-------------------- 20
---------- 10
----- 5
---------------- 16
-------- 8
---- 4
-- 2
- 1


If you are wondering about the proof for this wonderful bit of mathematics you can even request to see it with:

puts attempt1.proof

However, this method doesn't return the most satisfying answer.
