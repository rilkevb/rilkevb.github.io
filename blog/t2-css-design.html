Difference Between Relative, Absolute, and Fixed Positioning

10/21/14

Most elements on a webpage -- paragraphs, header, etc. -- are rectangular blocks of pixels. You can change the particular behaviors of these blocks by setting their display property to inline, inline-block, or block but they are still fundamentally blocks of pixels. Relative, absolute, and fixed positioning all move and pin these blocks in different ways.

Relative positioning seems to be one of the more commonly used types of positioning and it confused me for a while this week. The important thing to realize is that ‘relative position’ positions an element relative to where the element would naturally fall if no position property was set. You can see this by enabling ‘position: relative’ and then setting all the other positioning attributes to none. The element will be in the exact same place it was before you turned on any positioning.

Absolute positioning is the type to use when you want total control over where an element goes. You enable ‘position: absolute’ and then you set the positioning attributes to move the element exactly where you want it. Each position attribute value will move your block in relation to the next parent element with absolute positioning. If there is no such element the positioning will be relative to the html tag.

Fixed is the last type of positioning I learned about this week and seems to be one of the most novel. When ‘position: fixed’ is set, the element is positioned relative to the user’s screen. When you see nav bars which are glued at the top of your screen and float down with you as you scroll down, this has been accomplished by giving the element position fixed.
